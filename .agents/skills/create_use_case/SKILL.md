# Skill: create_use_case

## Purpose

Create an application use case aligned with clean architecture boundaries.

## Required Input

- Business goal.
- Actor(s) involved.
- Input data contract.
- Expected output contract.

## Output Format

- Use case name.
- Python module/class/function proposal.
- Typed input/output models.
- Execution flow (ordered steps).
- Error handling strategy.

## Execution Rules

1. Follow `.github/architecture.md`.
2. Apply `.github/standards.md`.
3. Respect `.github/domain-boundaries.md`.
4. Keep naming/docstrings in English.
